QUICK START ~</div><div class='line' id='LC48'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-quickstart*</div><div class='line' id='LC49'><br/></div><div class='line' id='LC50'>1. Introduction:</div><div class='line' id='LC51'><br/></div><div class='line' id='LC52'>&nbsp;&nbsp;Installation requires `Git` and triggers git clone for each configured</div><div class='line' id='LC53'>&nbsp;&nbsp;repository to `~/.vim/bundle/` by default. Curl is required for search.</div><div class='line' id='LC54'><br/></div><div class='line' id='LC55'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-windows*</div><div class='line' id='LC56'>&nbsp;&nbsp;If you are using Windows, see instructions on the Wiki</div><div class='line' id='LC57'>&nbsp;&nbsp;https://github.com/gmarik/Vundle.vim/wiki/Vundle-for-Windows.</div><div class='line' id='LC58'><br/></div><div class='line' id='LC59'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-faq*</div><div class='line' id='LC60'>&nbsp;&nbsp;If you run into any issues, please consult the FAQ at</div><div class='line' id='LC61'>&nbsp;&nbsp;https://github.com/gmarik/Vundle.vim/wiki</div><div class='line' id='LC62'><br/></div><div class='line' id='LC63'>2. Setup Vundle:</div><div class='line' id='LC64'>&gt;</div><div class='line' id='LC65'>&nbsp;&nbsp;git clone https://github.com/gmarik/Vundle.vim.git ~/.vim/bundle/Vundle.vim</div><div class='line' id='LC66'>&lt;</div><div class='line' id='LC67'>3. Configure bundles:</div><div class='line' id='LC68'><br/></div><div class='line' id='LC69'>&nbsp;&nbsp;Put this at the top of your `.vimrc` to use Vundle. Remove bundles you</div><div class='line' id='LC70'>&nbsp;&nbsp;don&#39;t need, they are for illustration purposes.</div><div class='line' id='LC71'>&gt;</div><div class='line' id='LC72'>&nbsp;&nbsp;&nbsp;&nbsp;set nocompatible              &quot; be iMproved, required</div><div class='line' id='LC73'>&nbsp;&nbsp;&nbsp;&nbsp;filetype off                  &quot; required</div><div class='line' id='LC74'><br/></div><div class='line' id='LC75'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; set the runtime path to include Vundle and initialize</div><div class='line' id='LC76'>&nbsp;&nbsp;&nbsp;&nbsp;set rtp+=~/.vim/bundle/Vundle.vim</div><div class='line' id='LC77'>&nbsp;&nbsp;&nbsp;&nbsp;call vundle#begin()</div><div class='line' id='LC78'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; alternatively, pass a path where Vundle should install plugins</div><div class='line' id='LC79'>&nbsp;&nbsp;&nbsp;&nbsp;&quot;call vundle#begin(&#39;~/some/path/here&#39;)</div><div class='line' id='LC80'><br/></div><div class='line' id='LC81'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; let Vundle manage Vundle, required</div><div class='line' id='LC82'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;gmarik/Vundle.vim&#39;</div><div class='line' id='LC83'><br/></div><div class='line' id='LC84'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; The following are examples of different formats supported.</div><div class='line' id='LC85'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; Keep Plugin commands between vundle#begin/end.</div><div class='line' id='LC86'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; plugin on GitHub repo</div><div class='line' id='LC87'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;tpope/vim-fugitive&#39;</div><div class='line' id='LC88'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; plugin from http://vim-scripts.org/vim/scripts.html</div><div class='line' id='LC89'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;L9&#39;</div><div class='line' id='LC90'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; plugin not on GitHub</div><div class='line' id='LC91'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;git://git.wincent.com/command-t.git&#39;</div><div class='line' id='LC92'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; git repos on your local machine (i.e. when working on your own plugin)</div><div class='line' id='LC93'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;file:///home/gmarik/path/to/plugin&#39;</div><div class='line' id='LC94'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; The sparkup vim script is in a subdirectory of this repo called vim.</div><div class='line' id='LC95'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; Pass the path to set the runtimepath properly.</div><div class='line' id='LC96'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;rstacruz/sparkup&#39;, {&#39;rtp&#39;: &#39;vim/&#39;}</div><div class='line' id='LC97'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; Avoid a name conflict with L9</div><div class='line' id='LC98'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;user/L9&#39;, {&#39;name&#39;: &#39;newL9&#39;}</div><div class='line' id='LC99'><br/></div><div class='line' id='LC100'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; All of your Plugins must be added before the following line</div><div class='line' id='LC101'>&nbsp;&nbsp;&nbsp;&nbsp;call vundle#end()            &quot; required</div><div class='line' id='LC102'>&nbsp;&nbsp;&nbsp;&nbsp;filetype plugin indent on    &quot; required</div><div class='line' id='LC103'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; To ignore plugin indent changes, instead use:</div><div class='line' id='LC104'>&nbsp;&nbsp;&nbsp;&nbsp;&quot;filetype plugin on</div><div class='line' id='LC105'>&nbsp;&nbsp;&nbsp;&nbsp;&quot;</div><div class='line' id='LC106'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; Brief help</div><div class='line' id='LC107'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; :PluginList          - list configured plugins</div><div class='line' id='LC108'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; :PluginInstall(!)    - install (update) plugins</div><div class='line' id='LC109'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; :PluginSearch(!) foo - search (or refresh cache first) for foo</div><div class='line' id='LC110'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; :PluginClean(!)      - confirm (or auto-approve) removal of unused plugins</div><div class='line' id='LC111'>&nbsp;&nbsp;&nbsp;&nbsp;&quot;</div><div class='line' id='LC112'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; see :h vundle for more details or wiki for FAQ</div><div class='line' id='LC113'>&nbsp;&nbsp;&nbsp;&nbsp;&quot; Put your non-Plugin stuff after this line</div><div class='line' id='LC114'><br/></div><div class='line' id='LC115'>4. Install configured bundles:</div><div class='line' id='LC116'><br/></div><div class='line' id='LC117'>&nbsp;&nbsp;Launch vim and run</div><div class='line' id='LC118'>&gt;</div><div class='line' id='LC119'>&nbsp;&nbsp;&nbsp;&nbsp;:PluginInstall</div><div class='line' id='LC120'>&lt;</div><div class='line' id='LC121'>&nbsp;&nbsp;To install from command line:</div><div class='line' id='LC122'>&gt;</div><div class='line' id='LC123'>&nbsp;&nbsp;&nbsp;&nbsp;vim +PluginInstall +qall</div><div class='line' id='LC124'><br/></div><div class='line' id='LC125'>=============================================================================</div><div class='line' id='LC126'>3. PLUGINS ~</div><div class='line' id='LC127'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins*</div><div class='line' id='LC128'><br/></div><div class='line' id='LC129'>3.1 CONFIGURING PLUGINS ~</div><div class='line' id='LC130'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-configure* *:Plugin*</div><div class='line' id='LC131'><br/></div><div class='line' id='LC132'>Vundle tracks what plugins you want configured by the `Plugin` commands in your</div><div class='line' id='LC133'>`.vimrc`. Each `Plugin` command tells Vundle to activate the script on startup</div><div class='line' id='LC134'>adding it to your |runtimepath|. Commenting out or removing the line will</div><div class='line' id='LC135'>disable the `Plugin`.</div><div class='line' id='LC136'><br/></div><div class='line' id='LC137'>Each `Plugin` command takes a URI pointing to the script. No comments should</div><div class='line' id='LC138'>follow on the same line as the command. Example:</div><div class='line' id='LC139'>&gt;</div><div class='line' id='LC140'>&nbsp;&nbsp;Plugin &#39;git_URI&#39;</div><div class='line' id='LC141'><br/></div><div class='line' id='LC142'>The `Bundle` command can optionally take a second argument after the URI. It</div><div class='line' id='LC143'>has to be a dictionary, separated from the URI by a comma. Each key-value pair</div><div class='line' id='LC144'>in the dictionary is a configuration option.</div><div class='line' id='LC145'><br/></div><div class='line' id='LC146'>The following per-script configuration options are available.</div><div class='line' id='LC147'><br/></div><div class='line' id='LC148'>The &#39;rtp&#39; option</div><div class='line' id='LC149'>----------------</div><div class='line' id='LC150'><br/></div><div class='line' id='LC151'>Specifies a directory inside the repository (relative path from the root of</div><div class='line' id='LC152'>the repository) where the vim plugin resides. It determines the path that will</div><div class='line' id='LC153'>be added to the |runtimepath|.</div><div class='line' id='LC154'><br/></div><div class='line' id='LC155'>For example:</div><div class='line' id='LC156'>&gt;</div><div class='line' id='LC157'>&nbsp;&nbsp;Plugin &#39;git_URI&#39;, {&#39;rtp&#39;: &#39;some/subdir/&#39;}</div><div class='line' id='LC158'><br/></div><div class='line' id='LC159'>This can be used with git repositories that put the vim plugin inside a</div><div class='line' id='LC160'>subdirectory.</div><div class='line' id='LC161'><br/></div><div class='line' id='LC162'>The &#39;name&#39; option</div><div class='line' id='LC163'>-----------------</div><div class='line' id='LC164'><br/></div><div class='line' id='LC165'>The name of the directory that will hold the local clone of the configured</div><div class='line' id='LC166'>script.</div><div class='line' id='LC167'><br/></div><div class='line' id='LC168'>For example:</div><div class='line' id='LC169'>&gt;</div><div class='line' id='LC170'>&nbsp;&nbsp;Plugin &#39;git_URI&#39;, {&#39;name&#39;: &#39;newPluginName&#39;}</div><div class='line' id='LC171'><br/></div><div class='line' id='LC172'>This can be used to prevent name collisions between plugins that Vundle would</div><div class='line' id='LC173'>otherwise try to clone into the same directory. It also provides an additional</div><div class='line' id='LC174'>level of customisation.</div><div class='line' id='LC175'><br/></div><div class='line' id='LC176'>The &#39;pinned&#39; option</div><div class='line' id='LC177'>-------------------</div><div class='line' id='LC178'><br/></div><div class='line' id='LC179'>A flag that, when set to a value of 1, tells Vundle not to perform any git</div><div class='line' id='LC180'>operations on the plugin, while still adding the existing plugin under the</div><div class='line' id='LC181'>`bundles` directories to the |runtimepath|.</div><div class='line' id='LC182'><br/></div><div class='line' id='LC183'>For example:</div><div class='line' id='LC184'>&gt;</div><div class='line' id='LC185'>&nbsp;&nbsp;Plugin &#39;mylocalplugin&#39;, {&#39;pinned&#39;: 1}</div><div class='line' id='LC186'><br/></div><div class='line' id='LC187'>This allows the users to include, with Vundle, plugins tracked with version</div><div class='line' id='LC188'>control systems other than git, but the user is responsible for cloning and</div><div class='line' id='LC189'>keeping up to date. It also allows the users to stay in the current version of</div><div class='line' id='LC190'>a plugin that might have previously been updated by Vundle.</div><div class='line' id='LC191'><br/></div><div class='line' id='LC192'>Please note that the URI will be treated the same as for any other plugins, so</div><div class='line' id='LC193'>only the last part of it will be added to the |runtimepath|. The user is</div><div class='line' id='LC194'>advised to use this flag only with single word URIs to avoid confusion.</div><div class='line' id='LC195'><br/></div><div class='line' id='LC196'>3.2 SUPPORTED URIS ~</div><div class='line' id='LC197'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-uris*</div><div class='line' id='LC198'><br/></div><div class='line' id='LC199'>`Vundle` integrates very well with both GitHub and vim-scripts.org</div><div class='line' id='LC200'>(http://vim-scripts.org/vim/scripts.html) allowing short URIs. It also allows</div><div class='line' id='LC201'>the use of any URI `git` recognizes. In all of the following cases (except</div><div class='line' id='LC202'>local) the &#39;https&#39; protocol is used, see Vundle&#39;s options to override this.</div><div class='line' id='LC203'><br/></div><div class='line' id='LC204'>More information on `git`&#39;s protocols can be found at:</div><div class='line' id='LC205'>http://git-scm.com/book/ch4-1.html</div><div class='line' id='LC206'><br/></div><div class='line' id='LC207'>GitHub</div><div class='line' id='LC208'>------</div><div class='line' id='LC209'>GitHub is used when a user/repo is passed to `Plugin`.</div><div class='line' id='LC210'>&gt;</div><div class='line' id='LC211'>&nbsp;&nbsp;Plugin &#39;gmarik/Vundle.vim&#39; =&gt; https://github.com/gmarik/Vundle.vim</div><div class='line' id='LC212'><br/></div><div class='line' id='LC213'>Vim Scripts</div><div class='line' id='LC214'>-----------</div><div class='line' id='LC215'>Any single word without a slash &#39;/&#39; is assumed to be from Vim Scripts.</div><div class='line' id='LC216'>&gt;</div><div class='line' id='LC217'>&nbsp;&nbsp;Plugin &#39;ctrlp.vim&#39; =&gt; https://github.com/vim-scripts/ctrlp.vim</div><div class='line' id='LC218'><br/></div><div class='line' id='LC219'>Other Git URIs</div><div class='line' id='LC220'>--------------</div><div class='line' id='LC221'>No modification is performed on valid URIs that point outside the above</div><div class='line' id='LC222'>URLs.</div><div class='line' id='LC223'>&gt;</div><div class='line' id='LC224'>&nbsp;&nbsp;Plugin &#39;git://git.wincent.com/command-t.git&#39;</div><div class='line' id='LC225'><br/></div><div class='line' id='LC226'>Local Plugins</div><div class='line' id='LC227'>-------------</div><div class='line' id='LC228'>The git protocol supports local installation using the &#39;file://&#39; protocol.</div><div class='line' id='LC229'>This is handy when developing plugins locally. Follow the protocol with an</div><div class='line' id='LC230'>absolute path to the script directory.</div><div class='line' id='LC231'>&gt;</div><div class='line' id='LC232'>&nbsp;&nbsp;Plugin &#39;file:///path/from/root/to/plugin&#39;</div><div class='line' id='LC233'><br/></div><div class='line' id='LC234'>3.3 INSTALLING PLUGINS ~</div><div class='line' id='LC235'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-install* *:PluginInstall*</div><div class='line' id='LC236'>&gt;</div><div class='line' id='LC237'>&nbsp;&nbsp;:PluginInstall</div><div class='line' id='LC238'><br/></div><div class='line' id='LC239'>Will install all plugins configured in your `.vimrc`. Newly installed</div><div class='line' id='LC240'>plugins will be automatically enabled. Some plugins may require extra steps</div><div class='line' id='LC241'>such as compilation or external programs, refer to their documentation.</div><div class='line' id='LC242'><br/></div><div class='line' id='LC243'>PluginInstall allows installation of plugins by name:</div><div class='line' id='LC244'>&gt;</div><div class='line' id='LC245'>&nbsp;&nbsp;:PluginInstall unite.vim</div><div class='line' id='LC246'><br/></div><div class='line' id='LC247'>Installs and activates unite.vim. You can use Tab to auto-complete known</div><div class='line' id='LC248'>script names. Note that the installation just described isn&#39;t permanent. To</div><div class='line' id='LC249'>finish, you must put `Plugin &#39;unite.vim&#39;` at the appropriate place in your</div><div class='line' id='LC250'>`.vimrc` to tell Vundle to load the plugin at startup.</div><div class='line' id='LC251'><br/></div><div class='line' id='LC252'>After installing plugins press &#39;l&#39; (lowercase &#39;L&#39;) to see the log of commands</div><div class='line' id='LC253'>if any errors occurred.</div><div class='line' id='LC254'><br/></div><div class='line' id='LC255'>3.4 UPDATING PLUGINS ~</div><div class='line' id='LC256'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-update* *:PluginUpdate* *:PluginInstall!*</div><div class='line' id='LC257'>&gt;</div><div class='line' id='LC258'>&nbsp;&nbsp;:PluginInstall!   &quot; NOTE: bang(!)</div><div class='line' id='LC259'>or &gt;</div><div class='line' id='LC260'>&nbsp;&nbsp;:PluginUpdate</div><div class='line' id='LC261'><br/></div><div class='line' id='LC262'>Installs or updates the configured plugins. Press &#39;u&#39; after updates complete</div><div class='line' id='LC263'>to see the change log of all updated bundles. Press &#39;l&#39; (lowercase &#39;L&#39;) to</div><div class='line' id='LC264'>see the log of commands if any errors occurred.</div><div class='line' id='LC265'><br/></div><div class='line' id='LC266'>3.5 SEARCHING PLUGINS ~</div><div class='line' id='LC267'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-search* *:PluginSearch*</div><div class='line' id='LC268'>&gt;</div><div class='line' id='LC269'>&nbsp;&nbsp;:PluginSearch</div><div class='line' id='LC270'><br/></div><div class='line' id='LC271'>Search requires that `curl` be available on the system. The command searches</div><div class='line' id='LC272'>Vim Scripts (http://vim-scripts.org/vim/scripts.html) for matching</div><div class='line' id='LC273'>plugins. Results display in a new split window. For example:</div><div class='line' id='LC274'>&gt;</div><div class='line' id='LC275'>&nbsp;&nbsp;PluginSearch foo</div><div class='line' id='LC276'><br/></div><div class='line' id='LC277'>displays:</div><div class='line' id='LC278'>&gt;</div><div class='line' id='LC279'>&nbsp;&nbsp;&quot;Search results for: foo</div><div class='line' id='LC280'>&nbsp;&nbsp;Plugin &#39;MarkdownFootnotes&#39;</div><div class='line' id='LC281'>&nbsp;&nbsp;Plugin &#39;VimFootnotes&#39;</div><div class='line' id='LC282'>&nbsp;&nbsp;Plugin &#39;foo.vim&#39;</div><div class='line' id='LC283'>&lt;</div><div class='line' id='LC284'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*:PluginSearch!*</div><div class='line' id='LC285'>Alternatively, you can refresh the script list before searching by adding a</div><div class='line' id='LC286'>bang to the command.</div><div class='line' id='LC287'>&gt;</div><div class='line' id='LC288'>&nbsp;&nbsp;:PluginSearch! foo</div><div class='line' id='LC289'><br/></div><div class='line' id='LC290'>If the command is run without argument:</div><div class='line' id='LC291'>&gt;</div><div class='line' id='LC292'>&nbsp;&nbsp;:PluginSearch!</div><div class='line' id='LC293'><br/></div><div class='line' id='LC294'>it will display all known plugins in the new split.</div><div class='line' id='LC295'><br/></div><div class='line' id='LC296'>3.6 LISTING BUNDLES ~</div><div class='line' id='LC297'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-list* *:PluginList*</div><div class='line' id='LC298'>&gt;</div><div class='line' id='LC299'>&nbsp;&nbsp;:PluginList</div><div class='line' id='LC300'><br/></div><div class='line' id='LC301'>Displays a list of installed bundles.</div><div class='line' id='LC302'><br/></div><div class='line' id='LC303'>3.7 CLEANING UP ~</div><div class='line' id='LC304'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-plugins-cleanup* *:PluginClean*</div><div class='line' id='LC305'>&gt;</div><div class='line' id='LC306'>&nbsp;&nbsp;:PluginClean</div><div class='line' id='LC307'><br/></div><div class='line' id='LC308'>Requests confirmation for the removal of all plugins no longered configured</div><div class='line' id='LC309'>in your `.vimrc` but present in your bundle installation directory</div><div class='line' id='LC310'>(default: `.vim/bundle/`).</div><div class='line' id='LC311'><br/></div><div class='line' id='LC312'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*:PluginClean!*</div><div class='line' id='LC313'>&gt;</div><div class='line' id='LC314'>&nbsp;&nbsp;:PluginClean!</div><div class='line' id='LC315'><br/></div><div class='line' id='LC316'>Automatically confirm removal of unused bundles.</div><div class='line' id='LC317'><br/></div><div class='line' id='LC318'>=============================================================================</div><div class='line' id='LC319'>4. INTERACTIVE MODE ~</div><div class='line' id='LC320'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-interactive*</div><div class='line' id='LC321'><br/></div><div class='line' id='LC322'>Vundle provides a simple interactive mode to help you explore new plugins</div><div class='line' id='LC323'>easily.  Interactive mode is available after any command that lists `Plugins`</div><div class='line' id='LC324'>such as PluginSearch, PluginList or Plugins. For instance:</div><div class='line' id='LC325'>&gt;</div><div class='line' id='LC326'>&nbsp;&nbsp;:PluginSearch! unite</div><div class='line' id='LC327'><br/></div><div class='line' id='LC328'>Searches for plugins matching &#39;unite&#39; and yields a split window with:</div><div class='line' id='LC329'>&gt;</div><div class='line' id='LC330'>&nbsp;&nbsp;&quot;Keymap: i - Install bundle; c - Cleanup; s - Search; R - Reload list</div><div class='line' id='LC331'>&nbsp;&nbsp;&quot;Search results for: unite</div><div class='line' id='LC332'>&nbsp;&nbsp;Plugin &#39;unite-scriptenames&#39;</div><div class='line' id='LC333'>&nbsp;&nbsp;Plugin &#39;unite.vim&#39;</div><div class='line' id='LC334'>&nbsp;&nbsp;Plugin &#39;unite-yarm&#39;</div><div class='line' id='LC335'>&nbsp;&nbsp;Plugin &#39;unite-gem&#39;</div><div class='line' id='LC336'>&nbsp;&nbsp;Plugin &#39;unite-locate&#39;</div><div class='line' id='LC337'>&nbsp;&nbsp;Plugin &#39;unite-font&#39;</div><div class='line' id='LC338'>&nbsp;&nbsp;Plugin &#39;unite-colorscheme&#39;</div><div class='line' id='LC339'><br/></div><div class='line' id='LC340'>To install a bundle, move your cursor to the Plugin of interest and then</div><div class='line' id='LC341'>select a command. To install &#39;unite.vim&#39; put your cursor on the line and</div><div class='line' id='LC342'>then push `i`. For a more complete list see |vundle-keymappings|. After</div><div class='line' id='LC343'>unite.vim is installed the `:Unite file` command should be available.</div><div class='line' id='LC344'><br/></div><div class='line' id='LC345'>Note: Interactive installation doesn&#39;t update your `.vimrc`.</div><div class='line' id='LC346'><br/></div><div class='line' id='LC347'>=============================================================================</div><div class='line' id='LC348'>5. KEY MAPPINGS ~</div><div class='line' id='LC349'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-keymappings*</div><div class='line' id='LC350'><br/></div><div class='line' id='LC351'>KEY | DESCRIPTION</div><div class='line' id='LC352'>----|-------------------------- &gt;</div><div class='line' id='LC353'>&nbsp;i  |  run :PluginInstall with name taken from line cursor is positioned on</div><div class='line' id='LC354'>&nbsp;I  |  same as i, but runs :PluginInstall! to update bundle</div><div class='line' id='LC355'>&nbsp;D  |  delete selected bundle (be careful not to remove local modifications)</div><div class='line' id='LC356'>&nbsp;c  |  run :PluginClean</div><div class='line' id='LC357'>&nbsp;s  |  run :PluginSearch</div><div class='line' id='LC358'>&nbsp;R  |  fetch fresh script list from server</div><div class='line' id='LC359'><br/></div><div class='line' id='LC360'>=============================================================================</div><div class='line' id='LC361'>6. OPTIONS ~</div><div class='line' id='LC362'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-options*</div><div class='line' id='LC363'>&gt;</div><div class='line' id='LC364'>&nbsp;&nbsp;let g:vundle_default_git_proto = &#39;git&#39;</div><div class='line' id='LC365'>&lt;</div><div class='line' id='LC366'>&nbsp;&nbsp;This option makes Vundle use `git` instead of `https` when building</div><div class='line' id='LC367'>&nbsp;&nbsp;absolute URIs. For example:</div><div class='line' id='LC368'>&gt;</div><div class='line' id='LC369'>&nbsp;&nbsp;&nbsp;&nbsp;Plugin &#39;sjl/gundo.vim&#39; -&gt; git@github.com:sjl/gundo.git</div><div class='line' id='LC370'><br/></div><div class='line' id='LC371'>=============================================================================</div><div class='line' id='LC372'>7. VUNDLE INTERFACE CHANGE ~</div><div class='line' id='LC373'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*vundle-interface-change* *:Bundle* *:BundleInstall!*</div><div class='line' id='LC374'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*:BundleUpdate* *:BundleSearch* *:BundleList* *:BundleClean!*</div><div class='line' id='LC375'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*:VundleInstall!* *:VundleUpdate* *:VundleSearch*</div><div class='line' id='LC376'>&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;*:VundleList* *:VundleClean!*</div><div class='line' id='LC377'><br/></div><div class='line' id='LC378'>&nbsp;&nbsp;In order to bring in new changes, Vundle is adopting a new interface.</div><div class='line' id='LC379'>&nbsp;&nbsp;Going forward we will support primarily the Plugin namespace, additionally</div><div class='line' id='LC380'>&nbsp;&nbsp;for convenience we will also alias some commands to the Vundle namespace.</div><div class='line' id='LC381'>&nbsp;&nbsp;The following table summarizes the interface changes.</div><div class='line' id='LC382'><br/></div><div class='line' id='LC383'>&nbsp;&nbsp;Deprecated Names  | New Names</div><div class='line' id='LC384'>&nbsp;&nbsp;-----------------------------</div><div class='line' id='LC385'>&nbsp;&nbsp;Bundle            | Plugin</div><div class='line' id='LC386'>&nbsp;&nbsp;BundleInstall(!)  | PluginInstall(!), VundleInstall(!)</div><div class='line' id='LC387'>&nbsp;&nbsp;BundleUpdate      | PluginUpdate, VundleUpdate</div><div class='line' id='LC388'>&nbsp;&nbsp;BundleSearch(!)   | PluginSearch(!), VundleSearch(!)</div><div class='line' id='LC389'>&nbsp;&nbsp;BundleClean       | PluginClean(!), VundleClean(!)</div><div class='line' id='LC390'>&nbsp;&nbsp;BundleList        | PluginList</div><div class='line' id='LC391'><br/></div><div class='line' id='LC392'>&nbsp;&nbsp;Note: The Bundle commands will be deprecated. You may continue using them,</div><div class='line' id='LC393'>&nbsp;&nbsp;but they may not get all future updates. For instance, we have enabled</div><div class='line' id='LC394'>&nbsp;&nbsp;comments on Plugin lines but not Bundle, since it requires a change in</div><div class='line' id='LC395'>&nbsp;&nbsp;command declaration.</div><div class='line' id='LC396'><br/></div><div class='line' id='LC397'>&quot; vim: set expandtab sts=2 ts=2 sw=2 tw=78 ft=help norl:</div></pre></div></td>
